# CodeStreak Application Context

## Overview
CodeStreak is a Chrome extension that helps developers track their coding streaks by connecting to their GitHub account. It uses GitHub OAuth for authentication and displays user profile information, repository count, and follower statistics in a popup interface.

## Architecture

### Frontend (Chrome Extension)
- **Manifest** (`manifest.json`): Defines extension metadata, permissions, and service worker
- **Background Service Worker** (`background.js`): Handles GitHub OAuth flow, token exchange, and state persistence
- **Popup UI** (`popup/`): 
  - `popup.html`: Structure and styling of the extension popup
  - `popup.js`: UI logic, state management, and user interactions
  - `popup.css`: Styling for the popup interface
- **Utilities** (`utils/api.js`): API communication helpers

### Backend (Inferred from code references)
- API endpoints at `http://localhost:5000`:
  - `/auth/github/token`: Exchanges GitHub OAuth code for access token
- Likely built with Node.js/Express (based on node_modules presence)

## Data Flow

### GitHub OAuth Process
1. User clicks "Continue with GitHub" button in popup
2. Popup sends `startGitHubOAuth` message to background service worker
3. Background worker:
   - Builds GitHub authorization URL with client_id, redirect_uri, and scope
   - Opens GitHub auth page in new tab
   - Stores tab ID in `chrome.storage.local` as `pendingAuthTabId`
4. User authenticates on GitHub and is redirected back to extension redirect URI
5. Background worker's `tabs.onUpdated` listener detects the redirect:
   - Verifies tab matches `pendingAuthTabId`
   - Extracts authorization code from URL parameters
   - Sends code to backend `/auth/github/token` endpoint
   - Receives access token from backend
   - Uses token to fetch user profile from `https://api.github.com/user`
   - Stores connection state (`isConnected: true`, `githubUser: user object`) in storage
   - Clears `pendingAuthTabId`
   - Closes auth tab
6. Popup (via polling or storage change detection) updates to connected state
7. Disconnect button removes stored credentials and resets UI

### State Persistence
- Uses `chrome.storage.local` to store:
  - `isConnected`: Boolean connection status
  - `githubUser`: User profile object (login, name, avatar_url, etc.)
  - `pendingAuthTabId`: Tab ID during OAuth flow
  - `authError`: Error message if authentication fails

## Development & Setup

### Prerequisites
- Google Chrome browser
- Local backend server running on `http://localhost:5000` with GitHub OAuth endpoints
- GitHub OAuth application configured with client ID matching `Ov23liQ0pxISITEzJA6b`

### Installation
1. Load unpacked extension in Chrome:
   - Navigate to `chrome://extensions`
   - Enable Developer mode
   - Click "Load unpacked"
   - Select the `extention` directory

### Backend Requirements
The extension expects a backend with:
- POST `/auth/github/token`: Accepts `{ code }` and returns `{ access_token }`
- Likely needs to:
  - Exchange code with GitHub using client secret
  - Return access token to extension
  - (Optional) Store tokens for future API calls

## Security Considerations
- Uses HTTP (not HTTPS) for localhost backend - acceptable for development
- GitHub OAuth flow keeps client secret on backend (not exposed to client)
- Tokens stored in chrome.storage.local (isolated to extension)
- Limited permissions: only requests `repo` scope from GitHub
